演目ID「$ARGUMENTS」のnote記事を生成してください。

## 手順

1. `tmp/catalog.json` から該当演目のエントリを探す
2. 演目の詳細JSONを `tmp/{演目ID}.json` から読む（なければR2のデータを参照）
3. 公演情報を確認する：
   - `https://kabukiplus.com/api/performances` から現在の公演一覧を取得
   - 演目名で検索し、現在上演中・近日上演の公演があれば劇場名・公演タイトル・日程を記事の「観に行くなら」セクションに反映する
   - API に演目レベルのデータがない場合は、WebFetch で `https://www.kabuki-bito.jp/theaters/` 等を確認し、該当演目の公演情報を探す
4. 以下のルールに従って記事を生成する
4. `note-articles/{演目ID}.md` に保存する
5. 保存後、frontmatter（`---`で囲まれた部分）と画像プロンプト（`<!-- 画像生成プロンプト ... -->`）を除いた**記事本文だけ**をクリップボードにコピーする。Windows環境ではUTF-8文字化け防止のため以下のコマンドを使うこと：
   ```
   sed -n '/^# /,/^<!-- 画像生成/{ /^<!-- 画像生成/d; p; }' note-articles/{演目ID}.md | powershell.exe -NoProfile -Command "[Console]::InputEncoding=[System.Text.Encoding]::UTF8; Set-Clipboard -Value (([System.IO.StreamReader]::new([Console]::OpenStandardInput(), [System.Text.Encoding]::UTF8)).ReadToEnd())"
   ```
6. 画像生成プロンプトはターミナルに表示する
7. 「クリップボードにコピーしました。noteに貼り付けてください」と伝える

## 記事ルール

### 対象読者
歌舞伎初心者。映画・ドラマきっかけで興味を持った人。初観劇の人。

### 構成（必須）

#### 冒頭メタ情報
```
---
title: 記事タイトル
enmoku_id: 演目ID
target_url: https://kabukiplus.com/kabuki/navi/enmoku/{演目ID}
created: {今日の日付 YYYY-MM-DD}
status: draft
---
```

#### 1. 導入（リード文）
- 今なぜこの演目が注目か（公演中・映画関連・季節など）
- 「初めてでも大丈夫」という安心感

#### 2. ざっくりあらすじ
- ネタバレしすぎない程度に3〜5段落
- 登場人物は主要3人程度に絞る
- 最後に「つまりこういう話」と一言まとめ

#### 3. ここが面白い！（見どころ）
- 初心者が「おっ」と思えるポイント2〜3個
- 有名なセリフや場面の紹介
- 「ここで拍手が起きます」等の体験描写

#### 4. 観に行くなら（観劇Tips）
- イヤホンガイドの推奨
- おすすめの席
- 予習不要であることの安心感

#### 5. 末尾誘導（必須・以下の形式で）
```
---

📖 **この演目をもっと詳しく**
登場人物の詳しい解説やあらすじの全文は「KABUKI PLUS+」で読めます。
👉 https://kabukiplus.com/kabuki/navi/enmoku/{演目ID}

📓 **観劇したら記録を残そう**
「KABUKI RECO」で観劇ログをつけると、あなただけの歌舞伎アルバムが完成します。
👉 https://kabukiplus.com/kabuki/reco

---
```

### 文体
- ですます調。親しみやすく、でも軽すぎない
- 専門用語には（）で読みと簡単な説明を添える
- 絵文字は見出しのみ。本文中は使わない
- 文字数：2000〜3000字

### SEOタイトル
「【初心者向け】{演目名}のあらすじ・見どころをやさしく解説」

## タイトル画像プロンプト

記事末尾に以下の形式でタイトル画像生成用プロンプトを出力する：

```
<!-- 画像生成プロンプト
Style: 和風・浮世絵風イラスト、テキストなし
Size: 1200x630
Prompt: {演目の象徴的モチーフを含む英語プロンプト}
-->
```

色調は和の配色（金・朱・藍・墨など）。演目の象徴的なモチーフ（桜、刀、月、雪など）を含める。

## 画像生成プロンプトのクリップボードコピー

記事保存後、Geminiチャット用の日本語画像生成プロンプトをクリップボードにコピーする。フォーマット：

```
以下の条件で画像を1枚生成してください。

用途：note記事のタイトル画像（{演目名}）
サイズ：横長（16:9）
スタイル：浮世絵風のイラスト調
モチーフ：{演目の象徴的なモチーフ}
色調：{和の配色}
雰囲気：{演目に合った雰囲気}
テキスト：なし（文字を入れないでください）
人物：なし
```

## noteハッシュタグ

記事末尾の誘導リンクの後に、noteのハッシュタグを出力する。以下の固定タグ＋演目固有タグを組み合わせる：

### 固定タグ（毎回必ず含める）
```
#歌舞伎 #kabuki #歌舞伎初心者 #伝統芸能 #日本文化 #観劇 #舞台 #KABUKIPLUS
```

### 演目固有タグ（演目に応じて追加）
- 演目名（例: `#曽根崎心中`）
- 作者名（例: `#近松門左衛門`）
- ジャンル（世話物なら `#世話物`、時代物なら `#時代物`）
- 関連キーワード（映画・ドラマ関連があれば追加。例: `#国宝` `#吉田修一`）
- 地歌舞伎関連の演目なら `#地歌舞伎`
