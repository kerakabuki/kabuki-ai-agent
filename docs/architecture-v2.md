## 1. プロジェクト概要

JIKABUKI×AI は、地歌舞伎（岐阜県郡上市・気良歌舞伎）を起点に、AIの力で歌舞伎の「知る・観る・演じる」体験を豊かにするプロジェクト。

| 項目 | 内容 |
|------|------|
| 運営 | 気良歌舞伎（けらかぶき） |
| 開発 | 気良歌舞伎事務局 |
| 本番URL | https://kabukiplus.com |
| Worker URL | https://kerakabuki.kerakabuki.workers.dev |
| 規模 | JS 約56,300行 / 72モジュール / コミット60回 |
| 技術基盤 | Cloudflare Workers / KV / R2 / Workers AI |
| AI | Gemini 2.5 Flash（主）+ Workers AI Llama 3.1 8B（副） |
| チャネル | LINE Bot + Web PWA（2面展開） |

**コンセプト — 3つの柱**

```
    地域文化
      ╲      ╱
        ╲  ╱
    ★ JIKABUKI × AI ★
        ╱  ╲
      ╱      ╲
  伝統芸能     デジタル化
```

- **地域文化** — 気良の地域に根付く伝統文化を未来へつなぐ
- **伝統芸能** — 歌舞伎という「表現の力」で地域を盛り上げる
- **デジタル化** — デジタルの新しい手段で保存と発信を可能にする

## 5. けらのすけ（AI）の設計

### 一言で

「歌舞伎の友達」— 教科書ではなく、会話で寄り添う存在。

### ペルソナ

| 項目 | 設定 |
|------|------|
| 名前 | けらのすけ |
| 出身 | 岐阜県郡上市の気良地区 |
| 一人称 | けらのすけ |
| 口調 | カジュアル（「だよ」「だね」「かな」） |
| 文字制限 | LINE: 300字 / Web: 500字 |
| 守備範囲外 | 「歌舞伎のことなら聞いてね」 |

### AI処理パイプライン（keraAIv2）

```
ユーザー入力
    │
    ▼
[1] キャッシュ確認 ─── ai_cache:{qNorm} → ヒットなら即返却（TTL 6h）
    │
    ▼
[2] レート制限チェック ─── gemini_rpm:{minute}、14 RPM上限
    │                         │
    │                    (超過時)→ Workers AI Llama 3.1 8B（フォールバック）
    ▼
[3] RAGコンテキスト構築 ─── buildKabukiContext()
    │   ├── 類義語展開（ひらがな → 正式名称）
    │   ├── 演目マッチング（3-gramスコアリング、最大2件）
    │   ├── 用語マッチング（3-gram + キーワード、最大3件）
    │   ├── FAQマッチング（質問の3-gram一致）
    │   ├── 公演情報（キーワード検出時、最大8件）
    │   └── 地歌舞伎情報（「気良」等のキーワード時）
    │   上限: 8,000文字
    │
    ▼
[4] システムプロンプト構築 ─── buildKeraSystemPrompt()
    │   ├── ペルソナ定義
    │   ├── RAGコンテキスト（あれば参照指示）
    │   └── ツール使用指示（なければ検索ツール使用）
    │
    ▼
[5] Gemini 2.5 Flash 呼び出し ─── callGemini()
    │   ├── temperature: 0.1（確定的応答）
    │   ├── maxOutputTokens: 1,500
    │   ├── topP: 0.9
    │   └── タイムアウト: 20秒
    │
    ▼
[6] レスポンス分岐
    ├── Function Call → executeKeraFunction() → 再度Gemini呼び出し
    └── テキスト応答 → 返却
    │
    ▼
[7] 後処理
    ├── 会話履歴保存（20メッセージ、1h TTL）
    └── 応答キャッシュ保存（6h TTL）
```

## 13. ターゲット整理

```
新規ユーザーの入口
      ↓
    DOJO（クイズ、体験）── バイラルポテンシャル
      ↓
    NAVI（学ぶ）→ LIVE（観る）→ RECO（記録する）
      ↓
    GATE ←→ KABUKI PLUS+（相互導線）
      ↓
    BASE / LABO（演者・運営者向け）
```

| モジュール | ターゲット | ユーザージャーニー上の役割 |
|-----------|-----------|------------------------|
| DOJO | 幅広い層 | 入口（バイラル）→ 興味喚起 |
| NAVI | 初心者〜中級者 | 知識習得 → ファン化 |
| LIVE | ファン全般 | 次の観劇へ導く |
| RECO | 定期的な観劇者 | 記録・共有 → リテンション |
| GATE | 一般の人 | 地歌舞伎との出会い |
| INFO | ファン・関係者 | 最新情報でつなぎ止め |
| BASE | 演者・事務局 | 運営効率化 → 継続支援 |
| LABO | 演者・開発者 | コンテンツ充実 → 品質向上 |
