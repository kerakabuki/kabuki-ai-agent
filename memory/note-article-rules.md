# note記事生成ルール

## 目的
歌舞伎初心者に演目の魅力を伝え、KABUKI PLUS+ への流入を増やす。

## 対象読者
- 歌舞伎に興味はあるが観たことがない人
- 映画・ドラマきっかけで歌舞伎に興味を持った人
- 初めて観劇する/したばかりの人

## 記事構成（必須セクション）

### 1. 導入（リード文）
- 今なぜこの演目が注目か（公演中・映画関連・季節など）
- 「初めてでも大丈夫」という安心感を与える一文

### 2. ざっくりあらすじ
- ネタバレしすぎない程度に、3〜5段落
- 登場人物は最小限（主要3人程度）に絞る
- 「つまりこういう話」と一言でまとめる

### 3. ここが面白い！（見どころ）
- 初心者が「おっ」と思えるポイント 2〜3個
- 有名なセリフや場面があれば紹介
- 「ここで拍手が起きます」等の観劇体験を想像させる描写

### 4. 観に行くなら（観劇Tips）
- 現在公演中なら劇場・日程・チケット情報
- イヤホンガイドの推奨
- 「初心者は○○席がおすすめ」

### 5. KABUKI PLUS+ 誘導（末尾）
- 演目の詳しい解説リンク → NAVI
- 観劇後の記録 → RECO
- 以下のフォーマットで：

```
---
📖 この演目をもっと詳しく知りたい方へ
登場人物の解説や詳しいあらすじは「KABUKI PLUS+」で読めます。
→ https://kabukiplus.com/kabuki/navi/enmoku/{演目ID}

📓 観劇したら記録を残そう
KABUKI PLUS+ の「KABUKI RECO」で観劇ログをつけると、
あなただけの歌舞伎アルバムが完成します。
→ https://kabukiplus.com/kabuki/reco
---
```

## 文体ルール
- ですます調
- 親しみやすく、でも軽すぎない（「歌舞伎って実は…」的な発見トーン）
- 専門用語には必ず（）で読みと簡単な説明を添える
- 絵文字は見出しのみ。本文中は使わない
- 文字数：2000〜3000字

## SEO意識
- タイトル：「【初心者向け】{演目名}のあらすじ・見どころをやさしく解説」
- タイトルに検索されやすいキーワードを含める
- 冒頭100文字に演目名と「あらすじ」「見どころ」を入れる

## タイトル画像
記事ごとにタイトル画像のプロンプトを生成する。

### 画像プロンプトのルール
- スタイル：和風・浮世絵風のイラスト調
- 要素：演目の象徴的なモチーフ（桜、刀、月など）
- テキスト要素なし（noteのタイトルが重なるため）
- アスペクト比：1200x630px（noteのOGP推奨サイズ）
- 色調：落ち着いた和の配色（金・朱・藍・墨など）
- 生成ツール想定：DALL-E / Gemini Imagen 等

### プロンプト出力形式
```
【タイトル画像プロンプト】
{英語のプロンプト}
サイズ: 1200x630
```

## 演目データの参照
- R2の演目JSONデータ（tmp/catalog.json, 個別JSON）を参照して正確性を担保
- あらすじ・登場人物はデータに基づくが、note記事用にかみ砕いて書き直す
- データの丸写しはしない

## 出力
- `note-articles/{演目ID}.md` に保存
- ファイル先頭にメタ情報を含める：

```
---
title: 記事タイトル
enmoku_id: 演目ID
target_url: https://kabukiplus.com/kabuki/navi/enmoku/{演目ID}
created: YYYY-MM-DD
status: draft
---
```
